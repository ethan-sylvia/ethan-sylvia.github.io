MP1 Writeup

Overview

For MP1 I decided to build a tool that can identify inaccessible HTML code. The three main functions of the tool is identifying images without alt text, identifying input elements without a corresponding label element, and identifying paragraph elements with a contrat ratio less than 3.5 with it's background color.

Once an inaccessible element is "identified", it receives a bright red border or a red text color. The main purpose of this tool is to help developers ensure websites they code are accessible. The following WCAGs were cited

  • https://www.w3.org/WAI/WCAG22/Understanding/labels-or-instructions.html
  • https://www.w3.org/WAI/WCAG21/Understanding/non-text-content.html
  • https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html

Process

Below roughly lays out the process I followed:

  • build the html popup with two different input types: checkboxes and a button
  • establish a connection between the popup script and the content script
  • build the methods to find inaccessible images, text, and forms

Issue Deep Dive

The issue I probably spent the longest time trying to figure out was how to grab the color styles from a element with the DOM. I needed to do this in my text contrast method in order to compare the colors between text elements and their background. I tried to access the color by using element.style.color, which didn't work. I spent like 1-2 hours googling until I found out element.style only refers to inline styling, and elements styled in a CSS file would need to be called with getComputedStyle(element).getPropertyValue('') instead.
